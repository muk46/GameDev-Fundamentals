# MVP 변환

## 좌표 공간

게임에서 물체의 위치를 표현하는 기준점이 여러 개 있다.

```
로컬 공간 → 월드 공간 → 뷰 공간 → 클립 공간 → 스크린 공간

    M          V          P
  (Model)    (View)   (Projection)
```

## 1. 로컬 공간 (Local / Object Space)

오브젝트 자기 자신이 기준. 모델링할 때의 좌표.

```
    검의 로컬 좌표:

    칼끝 (0, 2)
      |
      |
    손잡이 (0, 0)  ← 원점

    검 자체의 중심이 (0, 0)
```

## 2. 월드 공간 (World Space)

게임 세계가 기준. 모든 오브젝트가 공유하는 좌표.

```
    캐릭터가 (10, 5)에 있고 손에 검이 있으면:

    검의 칼끝 월드 좌표 = (10, 7)

    Model 행렬이 이 변환을 해줌!
    로컬 (0, 2) → 월드 (10, 7)
```

**Model 변환 = 이동 × 회전 × 스케일** (변환 행렬 조합)

## 3. 뷰 공간 (View / Camera Space)

카메라가 기준. 카메라 위치가 원점이 되도록 전체 월드를 이동/회전.

```
    월드:                      뷰 (카메라 기준):

    캐릭터 (10, 5)              캐릭터 (2, 0)
    카메라 (8, 5)     →        카메라 (0, 0) ← 원점
    적 (15, 5)                  적 (7, 0)
```

## 4. 클립 공간 (Clip Space)

카메라에 보이는 범위만 잘라내기 위한 공간.

```
    Projection 행렬이 하는 일:

    원근 투영:              직교 투영:
    가까이 → 크게            거리 상관없이 같은 크기
    멀리 → 작게

    3D 게임               2D 게임 / UI
```

## 5. 스크린 공간 (Screen Space)

최종적으로 모니터 화면의 픽셀 좌표.

```
    (0, 0) ────────── (1920, 0)
      |                    |
      |    게임 화면        |
      |                    |
    (0, 1080) ──── (1920, 1080)
```

## 전체 흐름

```
칼끝 로컬 좌표 (0, 2)
        │
        ▼  Model 행렬 (이동, 회전, 스케일)
        │
월드 좌표 (10, 7)
        │
        ▼  View 행렬 (카메라 기준으로 변환)
        │
뷰 좌표 (2, 2)
        │
        ▼  Projection 행렬 (원근/직교 투영)
        │
클립 좌표 → 스크린 좌표 (픽셀)
```

UE/Unity에서는 엔진이 알아서 해주지만, 면접에서 물어보고 셰이더 짤 때 직접 다뤄야 한다.

## 핵심 정리

| 변환 | 약자 | 하는 일 | 기준 |
|------|------|---------|------|
| Model | M | 로컬 → 월드 | 게임 세계 |
| View | V | 월드 → 뷰 | 카메라 |
| Projection | P | 뷰 → 클립 | 화면 |
