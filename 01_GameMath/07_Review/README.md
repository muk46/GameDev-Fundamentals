## 수업 1: 벡터

```
벡터 = 방향 + 크기

크기:    |V| = sqrt(x² + y² + z²)
정규화:  V / |V| → 크기 1, 방향만 남김
방향:    B - A = A에서 B로 가는 방향

게임 활용: 이동, 거리 비교, 추적 시스템
```

### 핵심 공식

```
크기:    |V| = sqrt(3² + 4²) = 5
정규화:  (3/5, 4/5) = (0.6, 0.8)
이동:    위치 += 정규화된방향 * 속도 * DeltaTime
```

## 수업 2: 내적 (Dot Product)

```
공식:    A · B = Ax*Bx + Ay*By + Az*Bz
기하학:  A · B = |A||B| cos(θ)

정규화된 벡터끼리 → cos(θ) 바로 나옴

결과 해석:
  > 0  → 앞쪽 (같은 방향)
  = 0  → 직각 (수직)
  < 0  → 뒤쪽 (반대 방향)

게임 활용: FOV 판별, 앞/뒤 판별
```

### FOV 판별 원리

```
cos(시야각/2) 를 기준값으로 설정
내적 결과 >= 기준값 → 시야 안
내적 결과 <  기준값 → 시야 밖
```

## 수업 3: 외적 (Cross Product)

```
2D:  Ax*By - Ay*Bx → 스칼라
  > 0 → 대상이 왼쪽
  < 0 → 대상이 오른쪽
  = 0 → 일직선

3D:  두 벡터에 수직인 새 벡터 생성 (오른손 법칙)
```

### 4방향 판별 (내적 + 외적 조합)

```
내적 > 0  &&  외적 > 0  → 앞-왼쪽
내적 > 0  &&  외적 < 0  → 앞-오른쪽
내적 < 0  &&  외적 > 0  → 뒤-왼쪽
내적 < 0  &&  외적 < 0  → 뒤-오른쪽
```

## 수업 4: 행렬

```
행렬 곱셈: 행 × 열, 교환법칙 성립 안 함 (AB ≠ BA)
단위 행렬(I): 곱해도 변하지 않는 행렬

변환 행렬 종류:
  스케일(S): 크기 변환
  회전(R):   각도만큼 회전
  이동(T):   위치 이동

적용 순서: S → R → T (SRT)
순서 바꾸면 결과 달라짐!
```

## 수업 5: MVP 변환

```
로컬 → 월드 → 뷰 → 클립 → 스크린

Model (M):      로컬 → 월드      오브젝트를 월드에 배치
View (V):       월드 → 카메라     카메라 기준으로 변환
Projection (P): 3D → 2D          원근/직교 투영
```

### 좌표 공간 정리

```
로컬 공간:   오브젝트 기준 (자기 중심이 원점)
월드 공간:   게임 세계 기준
뷰 공간:     카메라가 원점, 카메라 방향이 Z축
클립 공간:   -1 ~ 1 범위로 정규화
스크린 공간: 실제 화면 픽셀 좌표
```

## 수업 6: 쿼터니언

```
오일러 각: 직관적이지만 짐벌락 문제 있음
짐벌락:    축 2개가 겹쳐서 1개 자유도 상실 (90도 회전 시)

쿼터니언: Q = (x, y, z, w)
  x, y, z = 회전축 * sin(θ/2)
  w       = cos(θ/2)

항등 쿼터니언: (0, 0, 0, 1) = 회전 없음
```

### Slerp 보간

```
Slerp = 구면 선형 보간 (Spherical Linear Interpolation)

Lerp:   직선 경로 → 속도 불균일
Slerp:  구 표면 경로 → 속도 균일

Slerp(A, B, t)
  t = 0   → A
  t = 0.5 → 중간
  t = 1   → B
```

### 엔진 API 정리

| 기능 | UE | Unity |
|------|-----|-------|
| 쿼터니언 타입 | `FQuat` | `Quaternion` |
| 오일러→쿼터니언 | `FRotator(...).Quaternion()` | `Quaternion.Euler(x,y,z)` |
| 쿼터니언→오일러 | `.Rotator()` | `.eulerAngles` |
| Slerp | `FQuat::Slerp(A, B, t)` | `Quaternion.Slerp(A, B, t)` |
| 항등 | `FQuat::Identity` | `Quaternion.identity` |

### 약점 정리

1. **SRT 순서** — 두 번 틀림. 스케일 → 회전 → 이동. 이동이 마지막.
   - 이유: 이동 먼저 하고 회전하면 원점 기준으로 공전해버림
2. **FOV 반각** — 시야각 120도 → cos(반각 60°) = 0.5, 시야각 자체에 cos 적용하면 안 됨
3. **클립 공간 범위** — -1 ~ 1 (Projection 행렬이 정규화)

### 강점

- 벡터 연산 (크기, 정규화, 방향)
- 내적/외적 활용 (FOV, 4방향 판별)
- 쿼터니언 개념 및 UE API
