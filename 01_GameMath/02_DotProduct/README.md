# 내적 (Dot Product)

## 내적이란?

두 벡터를 넣으면 **숫자 하나(스칼라)**가 나오는 연산.

```
A · B = Ax * Bx + Ay * By
```

예) `A = (3, 4)`, `B = (2, 1)` → `3*2 + 4*1` = **10**

## 내적의 의미

```
A · B = |A| * |B| * cos(θ)     ← θ는 두 벡터 사이 각도
```

두 벡터가 **얼마나 같은 방향인지** 알 수 있다.

```
양수 (+)  →  같은 방향 (0° ~ 90°)

    A →
    B →

0         →  수직 (정확히 90°)

    A →
    B ↑

음수 (-)  →  반대 방향 (90° ~ 180°)

    A →
    B ←
```

## 정규화된 벡터의 내적 = cos(θ)

| 내적 결과 | 각도 | 의미 |
|-----------|------|------|
| 1 | 0° | 완전히 같은 방향 |
| 0.866 | 30° | 거의 같은 방향 |
| 0.5 | 60° | 비스듬히 같은 방향 |
| 0 | 90° | 수직 |
| -0.5 | 120° | 비스듬히 반대 |
| -0.866 | 150° | 거의 반대 |
| -1 | 180° | 완전히 반대 방향 |

## 게임에서의 활용

### 1. 적이 앞에 있는지 뒤에 있는지

```
        적
        *
       /
      /  내적 > 0 → 앞에 있음!
    플레이어 ──→ 전방(Forward)

    전방 벡터 · (적 방향) > 0 → 앞
    전방 벡터 · (적 방향) < 0 → 뒤
    전방 벡터 · (적 방향) = 0 → 옆
```

### 2. 시야각 (FOV) 체크

```
          시야각 60°
         /       \
        /    *적   \
       / (시야 안!) \
      /             \
    플레이어 ──→ 전방

    시야각 60° → 반각 30° → cos(30°) ≈ 0.866
    내적 결과 > 0.866 이면 시야 안에 있음!
```

## 실습: 시야각(FOV) 판별 시스템

→ [dot_fov.cpp](dot_fov.cpp)

```
플레이어 (0,0) → 전방 (1,0), FOV 90°

        적C (-1,2)
          \
           \
            적A (3,3)  ← 45° 경계선
           /
(0,0) ──→──── 적B (3,0)  ← 정면
           \
            적D (5,1)  ← 살짝 위
```

| 적 | 위치 | 각도 | 내적 | 결과 |
|----|------|------|------|------|
| A | (3,3) | 45° | 0.707 | 시야 경계 (안) |
| B | (3,0) | 0° | 1.0 | 시야 안 |
| C | (-1,2) | 뒤쪽 | 음수 | 시야 밖 |
| D | (5,1) | ~11° | ~0.98 | 시야 안 |

### 시야각 판별 공식

```
1. 방향 = Normalize(적 위치 - 플레이어 위치)
2. 내적 = Dot(전방 벡터, 방향)
3. 기준 = cos(시야각 / 2)
4. 내적 >= 기준 → 시야 안!
```

### 각도를 라디안으로 변환

```cpp
float radian = degrees * 3.14159265f / 180.0f;
// cos()에 넘길 때는 반드시 라디안으로!
```

## 퀴즈 오답 노트

- 내적 값과 각도의 관계: cos(θ)를 외울 필요 없이, **양수=같은방향, 0=수직, 음수=반대** 감만 잡으면 된다
- 내적 계산 시 반드시 **정규화된 벡터**를 써야 cos(θ) 값이 나온다 (안 하면 크기에 영향 받음)
- `(3,3)` 위치를 보고 "수직"이라 착각 → 실제로는 45°, 정규화하면 (0.707, 0.707)
