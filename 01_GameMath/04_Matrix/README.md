# 행렬 기초 (Matrix)

## 행렬이란?

숫자를 직사각형으로 나열한 것. 게임에서는 **변환(이동, 회전, 스케일)**을 표현하는 데 쓴다.

```
2x2 행렬:          3x3 행렬:

| a  b |          | a  b  c |
| c  d |          | d  e  f |
                  | g  h  i |
```

## 왜 행렬을 쓰나?

이동, 회전, 스케일을 행렬 하나로 합칠 수 있다.

```
따로 계산:               행렬로:
  1) 스케일 적용           변환 행렬 = 이동 * 회전 * 스케일
  2) 회전 적용             결과 = 변환 행렬 * 위치
  3) 이동 적용             → 행렬 하나로 끝!
  → 3번 연산
```

UE의 `FTransform`, Unity의 `Transform`이 내부적으로 행렬 연산을 한다.

## 행렬 곱셈

행 × 열로 계산.

```
| a  b |   | e  f |   | a*e+b*g  a*f+b*h |
| c  d | × | g  h | = | c*e+d*g  c*f+d*h |
```

예시:

```
| 2  1 |   | 1  0 |   | 2*1+1*3  2*0+1*4 |   | 5  4 |
| 3  0 | × | 3  4 | = | 3*1+0*3  3*0+0*4 | = | 3  0 |
```

### 중요: 곱셈 순서가 바뀌면 결과가 달라짐

```
A × B ≠ B × A    ← 교환법칙 안 됨!
```

게임에서의 의미:

```
이동 후 회전 ≠ 회전 후 이동

    이동 후 회전:    제자리에서 회전
    회전 후 이동:    원점 기준으로 회전 후 이동
    → 결과가 다름!
```

## 단위 행렬 (Identity Matrix)

곱해도 아무 변화 없는 행렬. 숫자의 1과 같은 역할.

```
| 1  0  0 |
| 0  1  0 |   × 어떤 행렬 = 어떤 행렬 그대로
| 0  0  1 |
```

## 변환 행렬

2D에서는 3x3 행렬을 사용 (3번째 행/열은 이동 표현을 위한 동차 좌표).

### 스케일 행렬

```
| Sx  0  0 |
|  0 Sy  0 |       (3, 4) * Scale(2, 2) = (6, 8)
|  0  0  1 |
```

### 이동 행렬

```
| 1  0  Tx |
| 0  1  Ty |       (3, 4) * Translate(5, 3) = (8, 7)
| 0  0   1 |
```

### 회전 행렬

원점 기준으로 θ만큼 반시계 방향 회전.

```
| cos(θ)  -sin(θ)  0 |
| sin(θ)   cos(θ)  0 |     (3, 4) * Rotate(90°) = (-4, 3)
|   0        0     1 |
```

각도는 반드시 **라디안으로 변환** 후 cos/sin에 넣어야 한다:
```cpp
float rad = degrees * 3.14159265f / 180.0f;
```

### 행렬 생성 시 주의점

- `Matrix3x3 Mat = {};` 로 **0 초기화** 먼저
- 필요한 자리만 값을 넣으면 됨
- 이동 행렬에서 **대각선 1** 빼먹으면 X, Y 값이 사라짐
