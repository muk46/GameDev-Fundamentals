# 행렬 기초 (Matrix)

## 행렬이란?

숫자를 직사각형으로 나열한 것. 게임에서는 **변환(이동, 회전, 스케일)**을 표현하는 데 쓴다.

```
2x2 행렬:          3x3 행렬:

| a  b |          | a  b  c |
| c  d |          | d  e  f |
                  | g  h  i |
```

## 왜 행렬을 쓰나?

이동, 회전, 스케일을 행렬 하나로 합칠 수 있다.

```
따로 계산:               행렬로:
  1) 스케일 적용           변환 행렬 = 이동 * 회전 * 스케일
  2) 회전 적용             결과 = 변환 행렬 * 위치
  3) 이동 적용             → 행렬 하나로 끝!
  → 3번 연산
```

UE의 `FTransform`, Unity의 `Transform`이 내부적으로 행렬 연산을 한다.

## 행렬 곱셈

행 × 열로 계산.

```
| a  b |   | e  f |   | a*e+b*g  a*f+b*h |
| c  d | × | g  h | = | c*e+d*g  c*f+d*h |
```

예시:

```
| 2  1 |   | 1  0 |   | 2*1+1*3  2*0+1*4 |   | 5  4 |
| 3  0 | × | 3  4 | = | 3*1+0*3  3*0+0*4 | = | 3  0 |
```

### 중요: 곱셈 순서가 바뀌면 결과가 달라짐

```
A × B ≠ B × A    ← 교환법칙 안 됨!
```

게임에서의 의미:

```
이동 후 회전 ≠ 회전 후 이동

    이동 후 회전:    제자리에서 회전
    회전 후 이동:    원점 기준으로 회전 후 이동
    → 결과가 다름!
```

## 단위 행렬 (Identity Matrix)

곱해도 아무 변화 없는 행렬. 숫자의 1과 같은 역할.

```
| 1  0  0 |
| 0  1  0 |   × 어떤 행렬 = 어떤 행렬 그대로
| 0  0  1 |
```
